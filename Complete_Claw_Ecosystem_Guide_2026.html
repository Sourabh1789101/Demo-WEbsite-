What is OpenClaw?

OpenClaw is an open-source, self-hosted personal AI assistant that runs on your own machine and connects to messaging apps you already use — WhatsApp, Telegram, Discord, Slack, iMessage, and 20+ others.

Unlike ChatGPT which forgets you when you close the tab, OpenClaw runs as a persistent background daemon. It can browse the web, read/write files, run shell commands, automate browsers, schedule cron jobs, and message you proactively — while remembering your preferences across months.

ℹ️
History Originally called MoltBot → ClawdBot → OpenClaw (Jan 2026). Created by Peter Steinberger. 310,000+ GitHub stars. MIT licensed. The founder joined OpenAI in Feb 2026; the project moved to an open-source foundation.

WhatsApp, Telegram, Discord, Slack, Signal, iMessage, Teams, Matrix, and more.

How It Works — The 6-Component Architecture

Every "Claw" variant follows this same fundamental pattern. Understanding it helps you use any of them effectively.

You → Channel (Telegram / WhatsApp) → Gateway → Agent Brain → Tools & Skills ↕ Memory System

1. Gateway — The Always-On Control Plane

A single Node.js process that routes messages. No AI intelligence lives here — it just connects your messaging channels, manages sessions, and forwards messages to the Agent. Runs as a background daemon (systemd / launchd) that survives reboots.

2. Agent Brain — The Reasoning Engine

Takes your message + assembled context and calls the LLM (Claude, GPT, local model, etc.). Uses a ReAct loop: reason → call tools → receive results → reason again → repeat until task is complete.

3. Tools — 26 Built-In Capabilities

File & System

bash shell commands · read/write/edit files · process management · git version control

Web & Browser

browser Chromium CDP automation · web_search · web_fetch page content · canvas visual workspace

Communication

message to channels · sessions_list/send/spawn · discord/slack platform-specific actions

Scheduling & Memory

cron recurring tasks · nodes companion apps · memory_write · gateway management

Every tool call passes through a 5-level policy chain before executing: global → provider → agent → session → sandbox.

4. Skills — Packaged Domain Expertise

Skills are directories containing a SKILL.md file with instructions. The runtime selectively injects only relevant skills per turn — not all 53 at once. You can create your own or ask the agent to build one for itself.

5. Memory — Filesystem as Source of Truth

  • Short-term: Daily logs in memory/YYYY-MM-DD.md
  • Long-term: Curated facts in MEMORY.md (preferences, projects, style)
  • Identity: SOUL.md (personality) + IDENTITY.md (who the agent is)
  • Retrieval: BM25 keyword + vector similarity. Indices rebuild from Markdown on startup.

6. Heartbeat & Cron — Proactive Intelligence

A heartbeat fires every 30 minutes. The agent reads HEARTBEAT.md and decides if action is needed — check emails, send reminders, monitor flights, etc. This makes it proactive, not just reactive.

Message Execution Flow — 10 Steps

This is what happens every single time you send a message:

  1. Channel adapter receives your message — WhatsApp (Baileys), Telegram (grammY), etc. Raw message is normalized.
  2. Gateway routes to session — DMs → main session. Groups → isolated session. Configurable per contact/channel.
  3. System prompt assembled — Reads IDENTITY.md + SOUL.md + TOOLS.md + relevant skills + memory files.
  4. Memory search — Queries for semantically similar past conversations to add useful context.
  5. LLM called with full context + tool registry — streams to your chosen model.
  6. Model responds — Runtime watches for tool calls (bash, browser, file read, etc.).
  7. Tool execution through 5-level policy chain. Sandboxed sessions run in Docker containers.
  8. Results fed back to model — Steps 6–8 repeat until no more tools are called.
  9. Response streamed to channel — Appears in your chat app in real time.
  10. Transcript persisted — Written to JSONL. Memory manager runs compaction if needed.

Workspace Files — The Agent's Home

Everything is plain Markdown on disk. Readable, editable, Git-backupable.

~/.openclaw/ ├── openclaw.json # Gateway config ├── .env # API keys (env vars) ├── agents/ │ └── <agentId>/ │ └── sessions/ │ └── *.jsonl # Conversation transcripts └── workspace/ # THE AGENT'S HOME ├── IDENTITY.md # Who the agent IS ├── SOUL.md # Personality & principles ├── AGENTS.md # Boot sequence / SOP ├── TOOLS.md # Your specific configs ├── USER.md # Who YOU are ├── MEMORY.md # Long-term memory ├── HEARTBEAT.md # Proactive checklist ├── memory/ │ └── YYYY-MM-DD.md # Daily logs └── skills/ └── <name>/ └── SKILL.md # Skill definition

SOUL.md — The Heart

# Soul
You are Molty, a helpful AI assistant.
## Core Principles
- Always be honest
- Ask before destructive operations
- Never delete files — move to trash
- Prefer concise responses

AGENTS.md — Boot Sequence

## Every Session
Before doing anything:
1. Read SOUL.md — who you are
2. Read USER.md — who you're helping
3. Read memory/YYYY-MM-DD.md (today + yesterday)
4. If MAIN SESSION: Also read MEMORY.md

HEARTBEAT.md — Proactive Checklist

- [ ] Check for urgent emails
- [ ] Monitor flight AA1234 for changes
- [ ] Remind user about dentist at 3pm

Pro Tip: Git-back your workspace Put ~/.openclaw/workspace in a private Git repo. This gives you backup, recovery, and a full history of how your agent's personality evolved.

OpenClaw — Installation & Setup

🚨
Security Warning As of March 2026: 8 critical CVEs (CVE-2026-25253 = one-click RCE), 42,665 exposed instances (93% with auth bypass), ~900 malicious skills on ClawHub (~20% of registry). Always use Docker + security hardening.

Prerequisites

  • Node.js 22+ (recommended: Node 24)
  • npm 9+ · Git (optional) · Docker (recommended)
  • An API key — Anthropic, OpenAI, Google, or Ollama for local
  • Windows: use WSL2 (PowerShell is unstable)
node --version    # v22.x+
npm --version     # 9.x+

Method 1 — Installer Script (Recommended)

curl -fsSL https://openclaw.ai/install.sh | bash

Method 2 — npm

npm install -g openclaw@latest
openclaw --version
openclaw onboard --install-daemon

Method 3 — Docker (Most Secure)

docker pull openclaw/openclaw:latest
docker run -d --name openclaw \
  -p 3000:3000 \
  -v ~/.openclaw:/root/.openclaw \
  -v ~/openclaw/workspace:/workspace \
  -e ANTHROPIC_API_KEY=sk-ant-your-key \
  openclaw/openclaw:latest

Method 4 — From Source

git clone https://github.com/openclaw/openclaw.git
cd openclaw && pnpm install && pnpm ui:build && pnpm build
pnpm openclaw onboard --install-daemon

Onboarding Wizard

  1. Security Warning — read and accept to continue.
  2. Quickstart or Manual — choose Quickstart.
  3. Select AI Provider — Anthropic, OpenAI, Google, or "Skip" for Ollama.
  4. Enter API Key — or better: export ANTHROPIC_API_KEY=sk-ant-...
  5. Choose Default Model — Claude Sonnet 4.6 for best cost/quality.
  6. Connect Channel — WhatsApp QR, Telegram bot token, or Discord.
  7. Workspace Created — SOUL.md, AGENTS.md, etc. auto-generated.

Essential Commands

openclaw doctor          # diagnose issues
openclaw status          # gateway status
openclaw dashboard       # open browser UI
openclaw logs --follow   # real-time logs
openclaw run "Summarize README.md in this directory"

Configuration

// ~/.openclaw/openclaw.json
{
  "agent": { "model": "anthropic/claude-sonnet-4-6" },
  "gateway": { "bind": "loopback" },
  "agents": { "defaults": { "sandbox": { "mode": "non-main" } } }
}

Security Guide — Do These First

🛡️
Non-Negotiable Security Steps
  1. Bind Gateway to localhost — "gateway": { "bind": "loopback" } (default 0.0.0.0 exposes everything)
  2. Env vars for API keys — export ANTHROPIC_API_KEY=... (never hardcode)
  3. Run in Docker — prevents access to SSH keys, credentials on host
  4. Verified skills only — ~20% of ClawHub is malware. Review source code
  5. Review MEMORY.md — "memory poisoning" embeds malicious instructions
  6. SOUL.md boundaries — "Never delete files" · "If uncertain, refuse"
  7. Never expose Control UI to internet — use SSH tunnel or VPN
  8. Sandbox non-main sessions — "sandbox": { "mode": "non-main" }
💡
Dedicated Machine Strategy Run your agent on a dedicated device (Mac Mini, VPS, Raspberry Pi) that doesn't hold sensitive files. Full agent power, contained risk. Cost: $5–12/month VPS or $0 for old hardware.

Frequently Asked Questions

Can I run offline?

Yes. ZeroClaw, PicoClaw, NanoBot support offline via Ollama. Need 8GB+ VRAM. Use 64k+ context models.

How is this different from ChatGPT?

ChatGPT = passive chat. OpenClaw = 24/7 agent with memory, file access, browser, cron, proactive heartbeat.

What about ClawdBot / MoltBot?

Same project, renamed twice. Uninstall old: npm uninstall -g clawdbot. Configs compatible.

Can the agent build its own skills?

Yes. Describe a task or point to a YouTube video. It creates a SKILL.md file in your workspace.

Is it safe?

With hardening: yes. Without: no. Chinese gov restricted it in state agencies. Follow the Security Guide.
