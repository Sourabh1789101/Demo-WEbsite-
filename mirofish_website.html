Overview

What is MiroFish?

A next-generation AI prediction engine that builds entire digital societies to forecast real-world outcomes — before they happen.

🌐
Swarm intelligence simulation
Thousands of autonomous AI agents interact, argue, and evolve — just like real human societies.
🧠
Knowledge graph foundation
GraphRAG extracts entities and relationships from your documents to build each agent's reality.
🔮
God's-eye view control
Inject new variables mid-simulation — "What if the Fed cuts rates by 50 bps?" — and watch the world reorganize.
📊
Structured prediction reports
ReportAgent synthesizes every interaction into narrative timelines, opinion maps, and forecasts.
🎙️
Interview individual agents
Ask any simulated agent why they held a position. Explore the reasoning behind emergent behavior.

🏆 The origin story

Guo Hangjiang — a 20-year-old student at Beijing University of Posts and Telecommunications — built MiroFish in just 10 days using "vibe coding" powered by AI assistants. He sent a rough demo video to Chen Tianqiao (founder of Shanda Group, former richest man in China). Within 24 hours, Chen committed 30 million yuan (~$4.1M) to incubate the project.

On March 7, 2026, MiroFish hit #1 on GitHub's global trending list — above OpenAI, Google, and Microsoft repositories.

Architecture

The 5-stage pipeline

Every simulation passes through five sequential stages — from raw document to interactive prediction world.

01
Knowledge graph construction
MiroFish reads your seed material using GraphRAG — Graph-based Retrieval Augmented Generation. It extracts all entities (people, organisations, events) and the relationships between them. Both individual and group memory structures are injected so agents can retain historical context across rounds.
GraphRAGEntity extractionMemory injectionZep Cloud
02
Environment generation
From the knowledge graph, the system generates agent personas — each with a unique personality, opinion bias, reaction speed, influence level, and social history. An Environment Configuration Agent sets the simulation rules and connects agents into a social network.
Persona generationSocial networkEnv config agentBehavioral logic
03
Parallel simulation execution
Agents interact simultaneously on two simulated platforms: a Twitter-like feed (fast, viral, influencer-driven) and a Reddit-like forum (threaded, community-driven). The OASIS engine supports up to 1 million agents and 23 types of social actions — posting, commenting, liking, following, reposting, muting, and more.
OASIS engineDual-platformUp to 1M agents23 social actions
04
Report generation
After all rounds complete, the ReportAgent uses three specialised tools: InsightForge (deep sub-query search), PanoramaSearch (full historical scope), and InterviewAgents (real-time IPC interviews). It produces a structured report covering narrative evolution, key moments, and counter-movements.
ReportAgentInsightForgePanoramaSearchInterviewAgents
05
Deep interaction
Users can talk directly with any individual agent ("What did you think about the announcement?"), ask the ReportAgent follow-up questions, or inject new variables into the simulation world from a God's-eye view — testing "what if a counter-statement is released?" scenarios in real time.
Agent interviewsGod's-eye viewVariable injectionScenario testing

Installation

Developer setup

Three ways to get MiroFish running locally — pick the one that fits your setup.

✅ Easiest method. Requires an LLM API key (free tiers available). No GPU needed.
1
Install prerequisites
Make sure Node.js 18+, Python 3.11–3.12, uv, and Git are installed. Verify with:
# Check each tool is installed
node -v           # Must be 18+
python --version  # Must be 3.11 or 3.12
uv --version      # pip install uv if missing
git --version
2
Clone the repository
Download MiroFish source code from GitHub to your computer.
git clone https://github.com/666ghj/MiroFish.git
cd MiroFish
3
Copy the environment file
Creates your personal config from the template.
cp .env.example .env
4
Get your API keys
You need two keys. Both have free tiers.
LLM: Qwen (bailian.aliyun.com, recommended) or OpenAI (platform.openai.com)
Memory: Zep Cloud (getzep.com) — free tier is enough for small experiments
5
Edit .env file
Open .env in any text editor and fill in your keys.
# LLM configuration
LLM_API_KEY=your_api_key_here
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus

# Memory system (Zep Cloud)
ZEP_API_KEY=your_zep_key_here
For OpenAI: change LLM_BASE_URL to https://api.openai.com/v1 and LLM_MODEL_NAME to gpt-4o-mini
6
Install all dependencies
One command installs both frontend (Node.js) and backend (Python). Auto-creates a virtual environment.
npm run setup:all
⚠️ If this fails: run npm cache clean --force then npm install --legacy-peer-deps
7
Launch the platform
Start both frontend and backend together.
npm run dev
Start separately if needed: npm run frontend (port 3000) · npm run backend (port 5001)
8
Open in your browser
http://localhost:3000
✅ If you see the MiroFish home screen — setup is complete. You're ready to run simulations!
🐳 Containerized method. Easier dependency management. Requires Docker Desktop installed.
1
Install Docker Desktop
Download from docker.com/products/docker-desktop. Make sure Docker is running (whale icon in taskbar) before proceeding.
2
Clone & configure .env
git clone https://github.com/666ghj/MiroFish.git
cd MiroFish
cp .env.example .env
# Edit .env with your LLM and Zep Cloud keys (same as cloud method Step 5)
3
Start with Docker Compose
First build takes 5–15 minutes as images download. Subsequent starts are instant.
docker compose up -d
Access the app at: http://localhost:5001 in Docker mode
4
Useful Docker commands
# Stop all services
docker compose down

# View live logs
docker compose logs -f

# Rebuild after code changes
docker compose up -d --build
Slow in China? Enable the commented mirror sources in docker-compose.yml to speed up image downloads.
⚠️ GPU with 16 GB+ VRAM strongly recommended. Use qwen2.5:14b on a 12 GB card. CPU-only works but is very slow. Uses the MiroFish-Offline fork — no API keys, no cloud, nothing leaves your machine.
1
Clone the offline fork
git clone https://github.com/nikmcfly/MiroFish-Offline.git
cd MiroFish-Offline
2
Configure .env (no API keys needed)
cp .env.example .env
# LLM — local Ollama
LLM_API_KEY=ollama
LLM_BASE_URL=http://localhost:11434/v1
LLM_MODEL_NAME=qwen2.5:32b

# Neo4j — local knowledge graph
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=mirofish

# Embeddings
EMBEDDING_MODEL=nomic-embed-text
EMBEDDING_BASE_URL=http://localhost:11434
3
Start all services with Docker
docker compose up -d
4
Pull the LLM models into Ollama
# Main LLM model (32B recommended, 14B for 12GB VRAM cards)
docker exec mirofish-ollama ollama pull qwen2.5:32b

# Embedding model (required)
docker exec mirofish-ollama ollama pull nomic-embed-text
5
Open in browser
http://localhost:3000
✅ Fully offline — all computation on your hardware, zero external calls.
Using MiroFish

Running your first simulation

Once the app is open in your browser, follow these 8 steps to run a prediction.

STEP 01
Click "New Prediction"
From the home screen, create a new simulation project. Choose the type: Public Opinion, Market Reaction, Story Prediction, or Policy Response.
STEP 02
Upload seed material
Paste or upload your document — news article, financial report, policy draft, research paper, or novel chapters. This is the reality agents will be built from.
STEP 03
Describe your prediction goal
Write your question in plain natural language. Be specific about the timeframe and what you want to observe. No special syntax needed.
STEP 04
Configure settings
Set simulation rounds (start with 20 or fewer), number of agents, and LLM model. Fewer rounds = faster and cheaper. Scale up once you know the workflow.
STEP 05
Start the simulation
Click Run. MiroFish builds a knowledge graph, generates agent personas, then runs agents across simulated Twitter and Reddit platforms simultaneously.
STEP 06
Wait for results
20 rounds · 50 agents ≈ 5–15 min. 40 rounds · 100 agents ≈ 20–45 min. Time depends on your LLM provider's speed and agent count.
STEP 07
Read the prediction report
ReportAgent synthesizes opinion shifts, dominant narratives, key influencer agents, emerging coalitions, and a final prediction summary.
STEP 08
Interact with the world
Interview any agent about their reasoning. Ask ReportAgent follow-ups. Inject new variables to test "what if?" scenarios from a God's-eye view.
💡 Example prediction prompts to try
"How will the public react to this layoff announcement over 30 days?"
"Predict how this policy will be debated across different stakeholder groups."
"What happens next in this story after chapter 80?"
"How will investors and media respond to this product launch?"
"If the Fed cuts rates by 50 bps, how will sentiment evolve over the next quarter?"
Troubleshooting

Common problems & fixes

Most issues come down to Python version, npm cache, or .env configuration. Here's how to resolve them.

Problem
Fix
Priority
npm install fails
Run npm cache clean --force then npm install --legacy-peer-deps
● High
Python version error
Must be exactly 3.11 or 3.12. Install pyenv: pyenv install 3.12.0 then pyenv local 3.12.0
● High
API call errors
Verify .env is saved correctly, check API key validity and account credits. Confirm the .env is loaded before running.
● High
Simulation is very slow
Reduce rounds to 20, reduce agent count, switch to a smaller model like gpt-4o-mini instead of gpt-4o.
● Med
Low RAM (under 16 GB)
Add MAX_CONCURRENT_AGENTS=5 to your .env file
● Med
Wrong model name error
Check your API provider's exact model string — naming formats differ per provider (e.g. qwen-plus vs qwen-turbo).
● Med
Port already in use
Another app is using port 3000 or 5001. Stop other apps or change the ports in your .env file.
● Low
Docker build takes too long
Normal on first run — images must download. Enable commented mirror sources in docker-compose.yml if in China.
● Low
