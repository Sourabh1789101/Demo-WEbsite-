What is NemoClaw?

NemoClaw is NVIDIA's open-source reference stack that runs OpenClaw always-on AI agents inside a hardened sandbox. It combines the NVIDIA OpenShell runtime with Nemotron models and policy-based egress control — all in a single install command.

Sandbox Image
~2.4 GB
Default Model
Nemotron 3 Super 120B
Install Time
1 command
⚠ Early Preview
NemoClaw is available in early preview starting March 16, 2026. Interfaces, APIs, and behavior may change without notice. Not yet production-ready — intended for feedback and experimentation.

Core Components

ComponentRoleType
nemoclaw CLI Primary user interface — orchestrates gateway, sandbox, inference, policy TypeScript / npm
Blueprint Versioned Python artifact for sandbox creation, digest verification, policy application Python artifact
OpenShell Sandbox Isolated container with Landlock + seccomp + network namespace isolation NVIDIA OpenShell
OpenClaw Agent The AI assistant running inside the sandbox, communicating via TUI or CLI OpenClaw runtime

Sandboxed Execution

Every agent runs with Landlock, seccomp, and network namespace isolation. No access granted by default.

Agent traffic routes through cloud-hosted Nemotron 3 Super 120B via build.nvidia.com, transparent to the agent.

Cloud API

Hot-reload

⚠ RAM Warning — OOM Risk
During sandbox image push, the Docker daemon, k3s, and the OpenShell gateway all run simultaneously alongside the export pipeline, buffering decompressed layers in memory. On machines with less than 8 GB RAM this can trigger the OOM killer. If you can't add RAM, configure at least 8 GB of swap as a workaround (at the cost of slower performance).

Installation

Step-by-Step Setup

1
Verify prerequisites
Ensure Docker is running, Node.js 20+ is available, and OpenShell is installed. Have your NVIDIA API key ready from build.nvidia.com.
2
Run the one-line installer
The script installs Node.js if absent, then launches the guided onboard wizard.
3
Onboard Wizard — Gateway & Provider
The wizard configures the OpenShell gateway, sets up your inference provider (NVIDIA cloud or local Ollama), and applies network + filesystem security policies.
4
Sandbox Creation
NemoClaw builds a Docker image (OpenClaw + plugin + entrypoint), creates the sandbox, and sets up port forwarding on port 18789. First build takes 2–5 minutes.
5
Connect and Chat
Connect to the sandbox and chat with your agent via the OpenClaw TUI or CLI.

Install Command

BASH — Run on Host Machine 
# Download and run the NemoClaw installer
$ curl -fsSL https://nvidia.com/nemoclaw.sh | bash
ℹ What the script does
Checks for Node.js (installs v20 if missing) → downloads the NemoClaw npm package globally → runs nemoclaw onboard wizard interactively → creates sandbox → configures inference → applies security policies → prints the connection summary.

PartLanguageRoleUpdate Cadence

Blueprint Lifecycle (4 Stages)

①
Resolve Artifact
Plugin downloads the versioned blueprint artifact, checks version compatibility.
②
Verify Digest
Blueprint artifacts are immutable and digest-verified before execution — supply chain safety.
③
Plan Resources
Blueprint determines which OpenShell resources to create or update: gateway, inference providers, sandbox, network policy.
④
Apply via OpenShell CLI
Blueprint calls OpenShell CLI commands to create the sandbox and configure each resource. Agent starts inside with all controls in place.

Sandbox & Security Policy

Every sandbox starts with a strict baseline policy defined in openclaw-sandbox.yaml. Four enforcement layers protect the host:

Network Egress

Blocks unauthorized outbound connections. Only endpoints listed in policy are allowed. Unknown hosts are blocked and surfaced in the TUI for operator approval.

⟳ Hot-reloadable

Filesystem

Prevents reads/writes outside /sandbox and /tmp. Agent can write to /sandbox and /tmp. All other system paths are read-only.

🔒 Locked at creation

System Calls (seccomp)

Blocks privilege escalation and dangerous syscalls from within the sandbox. Prevents container escape attempts.

🔒 Locked at creation

Inference Routing

Reroutes all model API calls to controlled backends (NVIDIA cloud or local Ollama/vLLM). Inference requests never leave the sandbox directly.

⟳ Hot-reloadable
ℹ Network Approval Flow
When the agent tries to reach an unlisted host, OpenShell blocks the request and surfaces it in the TUI for operator approval. Approved endpoints persist for the current session but are not saved to the baseline policy file permanently.

Customizing Network Policy

YAML — openclaw-sandbox.yaml (example) 
network:
  egress:
    # Pre-approved trusted domains
    allow:
      - api.github.com
      - pypi.org
      - registry.npmjs.org
    # Unknown hosts are blocked + surfaced for approval
    default: deny

Sandbox Management Commands

BASH
# Check sandbox status
$ nemoclaw my-assistant status
$ openshell sandbox list

# Follow live logs
$ nemoclaw my-assistant logs --follow

# Approve or deny a pending egress request (interactive TUI)
$ nemoclaw my-assistant connect
# Then approve/deny from within the TUI

Inference Configuration

How Inference Works

Inference requests from the agent never leave the sandbox directly. OpenShell intercepts every inference call and routes it to the configured provider. You can switch models at runtime without restarting the sandbox.

Available Providers

ProviderDefault ModelStatusAPI Key
NVIDIA Cloud (build.nvidia.com) Nemotron 3 Super 120B Production Required
Ollama (local) nemotron-3-nano (auto) Experimental Not needed
vLLM (local) Any local vLLM model Experimental Not needed
⚠ macOS Local Inference
On macOS, local inference options (Ollama, vLLM) are experimental and also depend on OpenShell host-routing support in addition to the local service itself being reachable on the host.

Switch Inference Provider at Runtime

BASH
# List available providers
$ openshell inference list

# Switch to local Ollama provider
$ openshell inference use ollama-local

# Switch back to NVIDIA cloud
$ openshell inference use nvidia-cloud

# No sandbox restart required

Troubleshooting

ℹ Two Error Layers
Errors may originate from either NemoClaw or the OpenShell layer underneath. Run nemoclaw <n> status for NemoClaw-level health and openshell sandbox list to check the underlying sandbox state.
OOM killer terminates setup during sandbox image push

The combined memory usage of Docker daemon, k3s, and OpenShell gateway during image export can exceed available RAM on machines with <8 GB.

BASH — Add 8 GB Swap
$ sudo fallocate -l 8G /swapfile
$ sudo chmod 600 /swapfile
$ sudo mkswap /swapfile
$ sudo swapon /swapfile
Port 8080 or 18789 already in use

The NemoClaw gateway requires ports 8080 and 18789 to be free. Check what's using them and stop the conflicting process.

BASH
$ sudo lsof -i :8080
$ sudo lsof -i :18789
Network request blocked — agent can't reach external host

The sandbox's strict baseline policy blocks all unknown egress. When blocked, the request appears in the TUI for operator approval. To pre-approve a domain, add it to the network policy YAML and hot-reload.

BASH — Approve in TUI
$ nemoclaw my-assistant connect
# Look for blocked request notification in TUI and approve/deny
Local inference not detected on macOS

On macOS, local Ollama/vLLM requires OpenShell host-routing support in addition to the model server running on the host. Ensure OpenShell's host-routing feature is enabled and the local service is reachable before running nemoclaw onboard.

Setup fails on platform — manual workarounds needed

NemoClaw is in early preview — setup may require manual workarounds on some platforms. If you encounter a blocker, file an issue at github.com/NVIDIA/NemoClaw/issues or join the Discord community.

Uninstall

To remove NemoClaw and all resources created during setup, run the uninstall script from the terminal outside the sandbox:

BASH — Interactive Uninstall 
$ curl -fsSL https://raw.githubusercontent.com/NVIDIA/NemoClaw/refs/heads/main/uninstall.sh | bash
BASH — Non-Interactive (skip confirmation) 
$ curl -fsSL https://raw.githubusercontent.com/NVIDIA/NemoClaw/refs/heads/main/uninstall.sh | bash -s -- --yes
